# Project State

## Current state summary (≤200 words)
WebDrop v2 is at **version 1.0.93**. The physical-proximity path now uses a server-persisted runtime policy: score weights, minimum score, late-tap grace, acoustic window, and bump-time slop are editable from the admin **Settings** tab and snapshotted per session. Defaults are a 6,000 ms late-tap grace, 6,000 ms acoustic window, and 4,000 ms match slop. The admin **Test cases** tab assigns live devices to intended pairs, records bounded diagnostic evidence, and keeps local run history tied to the exact policy revision. Capacity remains 100 participants / up to 50 pairs globally, with a configured six-device cohort cap. The core static PWA, WebRTC transfer pipeline, storage ladder, English/Japanese UI, and Azure signaling backend remain intact.

At the start of this session the working tree was effectively clean — the only pre-existing change in git status was an untracked `.cursor/mcp.json`. No application source files were mid-edit. Unit (`node --test`) and Playwright e2e suites exist; notable gaps: no coverage gate, no dedicated test for the data-channel transfer protocol, and live/relay e2e specs require the external server (see `tests.md`).

This `agent/` knowledge base was generated today from the real code as the standardized agent context for the repo.

## Recent changes
- **2026-07-01 — Runtime proximity tuning + repeatable multi-device test console (v1.0.93):** Added `RuntimeProximityPolicy` with validation, atomic JSON persistence, authenticated `PUT /api/proximity-policy`, and session-level policy snapshots. Late partner admission now waits for the full configurable grace (default 6 s), then starts promptly once another device joins; acoustic duration and match slop come from the same snapshot. Client scoring and ceremony timing consume server tuning. Moved tuning to a dedicated admin **Settings** tab. Added a **Test cases** tab with seven researched scenarios, live device Pair A/B assignment, start/stop recording, policy revision capture, event de-duplication, correct/wrong-pair counts, evidence rates, medians, and local history. Added English/Japanese labels, responsive fixes, unit/backend/e2e coverage, and service-worker precache for `test-runs.js`. `npm run verify` is green (49/49), backend tests are green (54/54), and the targeted admin Playwright suite is green (5/5). Browser QA proved Settings, a start/stop run with four simulated devices, and 390 px layout without horizontal overflow. See ADR-0009.
- **2026-06-30 — Tiles-as-progress HUD, swipe polish, live profiles (v1.0.92):** Three fixes. **(A) Transfer HUD = the tiles.** Removed the separate gradient progress bar; the island tile-wave itself now visualizes progress. `TileWave` gained `setTransferMode(on)`/`setProgress(ratio)`; in the per-tile draw loop, tiles whose `(sweepCol+1)/cols > progress` render as a dim, desaturated "remaining track" (no glow) while filled tiles keep the animated sweep — so the lit region grows L→R (send) / R→L (receive). Honored in the reduced-motion/static path too (`#drawStatic` + `#repaintIfStatic`). `DynamicIsland.syncWave` toggles transfer mode; `paintTransferProgress` drives `wave.setProgress` and no longer writes the meter. Deleted `.webdrop-island__meter` (CSS + HTML), kept the wave full-opacity during transfer, and restyled `.webdrop-island__transfer` as a full-height flex column (percent/label top, filename bottom, tiles through the middle). **(B) Swipe-up-to-send polish** (`css/sheets.css`): softened the thumb's blue shadow, trimmed `.swipe-control--vertical` height (132→110px, maxDistance ≈48px) so there's no big empty blue runway, lightened the `::before` affordance, and lifted the "Swipe up to send" label cleanly above the thumb. Gesture math (`bindSwipe`) unchanged. **(C) Live profile propagation:** changing avatar/ring/name now broadcasts `{type:"profile:update", profile}` over the open data channel (new `WebRtcTransport.sendControlMessage` + forwarded `control` event; `DataChannelTransferProtocol.sendControl` reused, default-case `control` emit) so the connected peer's store record + connected-screen avatar + island peer avatar refresh live (`DynamicIsland.updatePeerProfile`). Best-effort presence re-announce for nearby devices via `WebSocketSignalingAdapter.updateProfile(self)` (re-sends `client:hello` with cached capabilities; `MockSignalingAdapter` no-ops). 250 ms debounce + change-detection avoid spam. Bumped to **1.0.92**; `npm run check` + 45 unit tests green. See ADR-0007, ADR-0008.
