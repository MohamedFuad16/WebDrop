# Project State

## Current state summary (≤200 words)
WebDrop v2 is at **version 1.0.96**. The physical-proximity path now uses a server-persisted runtime policy: score weights, minimum score, late-tap grace, acoustic window, and bump-time slop are editable from the admin **Settings** tab and snapshotted per session. Defaults are a 6,000 ms late-tap grace, 6,000 ms acoustic window, and 4,000 ms match slop. The admin **Test cases** tab assigns live devices to intended pairs, records bounded diagnostic evidence, and keeps local run history tied to the exact policy revision. Capacity remains 100 participants / up to 50 pairs globally, with a configured six-device cohort cap. The core static PWA, WebRTC transfer pipeline, storage ladder, English/Japanese UI, and Azure signaling backend remain intact.

At the start of this session the working tree was effectively clean — the only pre-existing change in git status was an untracked `.cursor/mcp.json`. No application source files were mid-edit. Unit (`node --test`) and Playwright e2e suites exist; notable gaps: no coverage gate, no dedicated test for the data-channel transfer protocol, and live/relay e2e specs require the external server (see `tests.md`).

This `agent/` knowledge base was generated today from the real code as the standardized agent context for the repo.
