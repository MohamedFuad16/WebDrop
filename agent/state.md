# Project State

## Current state summary (≤200 words)
WebDrop v2 is at **version 1.0.90** — a mature, working static PWA for AirDrop-style proximity file transfer. Both signaling adapters (production WebSocket + in-memory mock) are implemented, as is the full pipeline: proximity ceremony (ultrasound + motion) with QR fallback, WebRTC data-channel transfer (manifest + chunked + acked, 500 MB cap), the three-tier storage ladder, the orbital UI + dynamic island, i18n (en/ja), and an operations/admin dashboard. Production config points at the Azure signaling server (`runtime-config.js`). A separate Node backend lives in `azure cloud server/`.

At the start of this session the working tree was effectively clean — the only pre-existing change in git status was an untracked `.cursor/mcp.json`. No application source files were mid-edit. Unit (`node --test`) and Playwright e2e suites exist; notable gaps: no coverage gate, no dedicated test for the data-channel transfer protocol, and live/relay e2e specs require the external server (see `tests.md`).

This `agent/` knowledge base was generated today from the real code as the standardized agent context for the repo.
